Overview

Learn how to make a mobile city building Unity game with the city builder game starter kit available on the Unity Asset Store and also from CityBuildingKit.com. This kit includes a basic framework for buildings, characters, HUD and gameboard actions. The following Unity tutorial documentation describes how to test on your mobile device out of the box, understand the HUD and edit buildings and characters plus unity script references for your game development. Get started with this starter kit for Unity Mobile games.

Requirements: NGUI and 2D Toolkit are required to run the starter kit. As per NGUI's team instructions, NGUI should be imported in a new empty scene, not directly in the game scene (throws a few internal NGUI errors otherwise). Safer to apply this to 2dToolKit as well. 

Video Demo

Unity Tutorial: Test out of the box on your iPhone (Mac)

1. Open the starter kit in Unity. As mentioned above, NGUI should be imported in a new empty scene, not directly in the game scene (throws a few internal NGUI errors otherwise). Safer to apply this to 2dToolKit as well.

2. Connect your iPhone and make sure Xcode is downloaded and installed on your Mac. If you haven't already, check that you've run a test app from Apple correctly on your iOS device and have your team keychain configured. When ready, open the project in Unity.

3. Click File > Build Settings

4. Click iOS and then Build and Run

5. Wait a moment while Unity builds the app, launches Xcode, and installs on your phone.              

All-inclusive Support (Android, iPhone, iPod, iPad)

This Starter Kit has been tested and deployed with all iOS devices (iPhone 5, iPod, iPad 1). Also tested on Android (Samsung Galaxy S), desktop and web deployment. All with great performance. With some devices, you may notice blanks between titles. This is a NGUI/Unity glitch; going around it requires modifications for the texture filtering and/or position snap in Unity (already done many times, but the position appears to drift a little, just enough to make it visible on Android). It appears that it behaves differently on different platforms - at this point, it's obviously not visible on either iPhone, iPod, MAC or PC.




Unity Starter Kit Mobile Menu Interface (UI)

Like all 2D isometric mobile games, the menu options are accessable from large buttons on the interface. From top to bottom, these are:

- XP Bar
 Value increases as buildings are produced and XP is gained (Standard in any 2D Building games)

- Developer Controls
 Use these arrows or zoom buttons for movement on non-iOS devices such as a desktop test. Hide in gameplay after development (Useful with this Unity Development Kit)

- Builders Available
 One building in construction is possible for every builder available and not currently used. On tap opens the in-game purchase menu to purchase additional builders. (Seen in other Unity social games)

- Resources / Soft-Currency #1 and #2  
Gold and Mana. Replace with your own terminology like Wood, Stone, Elixir, Silver, etc. Starts with predefined values as specified in the XML of this casual game template. More can be purchased or generated in the game with resource producing buildings included in the Unity game starter kits.

- Gems / Jewels / Hard-Currency Used in Unity Social Mobile Games  
Click to buy more, perfect for your mobile game commercial template. Used to speed up time sensitive tasks. Cannot be generated in the game like soft-currencies. Require the use of funds to get. Click to buy more in the in-game menu. (The best asset template to monetize what users buy)

- Competitions  
Prefab asset coming soon. Requires players to produce certain buildings or characters before they can compete. A way to expend the character's developed and turn this into a true Unity social game.

- Achievements
Coming soon. Focuses players on doing certain tasks to get XP rewards.

- Building Options
These show only when a building is selected. The example buttons include Upgrade to improve a building, Move, and Delete options.

- Settings Control in-game settings like sound and music in this Unity Asset

- Building & Character Shop
The central part of the game for purchasing and shopping for new items in this template game starter kit.
 

Navigation

Use the arrows and zoom buttons in the top left of the game to nagivate (Developer Controls) Hide in gameplay after development.


Saving and Loading

While you're developing, the starter kit will always start with an empty map, as if you were a new user. Save and load your gameplay during testing using the Save and Load Game features. Open the Settings menu on the right side and press save. Then close the game. Building locations, resource values, and construction times will be saved. Loading: Work only in an empty map. Start the game and press Load.

How to Setup In-App Purchases

In-App Purchases are setup as a demo right now in HardPriceProc.cs and don't connect directly to iOS or Android In-App Purchase routines. You will need to write your own In-App Purchase code or get an asset such as Prime31 or Unibill. Test by clicking the plus button (+) next to the gems or construction workers in the menu buttons on the main screen.

The HardPriceProc.cs script processes the total amount based on the info in HardPrice.xml (see below for XML excerpt). You can add and substract a total value. The totalPrice float variable must reach the solution you've picked for real online purchases. Create and test with this advice or get an asset such as Prime31 or Unibill. Remember to condition the resources increase in the main interface after the online purchase has been completed, and not just opening and canceling it.





Snapping Buildings to Toolkit2D Tile Grid

If you make your own map, using Toolkit2D - here's how to make your buildings snap to the tilemap grid:

1. Adjust the grid size in BuildingCreator.cs, based on the size of one map tile; below is what we use - your dimensions are probably different:

                    private int gridx = 256; //necessary to adjust the middle screen "target" to the exact grid X position 

private int gridy = 181; //necessary to adjust the middle screen "target" to the exact grid Y position
2. Then position the entire background tile map exactly in the center of the screen , based on number of tiles+tile size. Camera points in the middle.

HardPrice.xml (Store Items to Purchase)
In this file you can set the prices for each item to be purchased in the store and the amounts.
- Builders: Dobbit is the name of the builders (and the cost per additional builder to support multiple constructions at one time)
- Hard-Currency: Crystals is the name of the one hard-currency supported.
- Soft-Currency: Gold is the first soft-currency and Mana is the second. You can rename the soft-currencies to anything you wish.

Object Layers

The map and buildings/vegetation is built in several layers, to accommodate moving units, so they appear properly in front or behind objects. 

From back to front:

  • 1. Base layer - ocean, sand, grass
  • 2. Square patches of grass, on which the buildings reside
  • 3. Tall vegetation/buildings/rocks are built from 2 layers - base (background) and top (foreground); bases will also have colliders, to prevent units from getting too close and intersecting the sprite 'base edges'
  • 4. Sea units - although in front of the ocean layer, they will carry a 'patch' of water with them while moving
  • 5. Ground units - in front of building bases, but in the back of building tops, to be able to go 'behind' them
  • 6. Air units - the red gargoyle - while in the air, they appear in foreground on top of everything else

(The characters in the image are concept art only - not part of animation sheets.)

Expansions Coming Soon

Competitions.xml
Add in your own competitions to win resoucres and XP. Each competition can cost soft-currency like gold or hard-currency like gems. Also you can set the chance of winning a competition and also the cost of units. Each competition consumes the resources/units you require and after the time delay you set will award the user the prize you set depending on the chance of winning.

Creatures.xml
Like units, these are characters in the game under a special category. Like Clash of Clan's heros.

Artifacts.xml
These are special items, like decoration, players can purchase and put around the map. No special use other than decoration.

Research.xml
Certain buildings can be improved with research to unlock specific features, units, or competitions. 




 

NGUI and 2D Toolkit Updates

As NGUI and 2D Toolkit updates are released, minor changes to the kit will be required if you update your existing NGUI and 2D toolkit assets. If you do have to update, test and check your game for any substantial changes that have occured because of the update.
